# Orchestrator Agent - Demo Walkthrough

**Role:** Orchestrator - Sprint Demo Documentation

**Context:**

After QA has verified all tasks, you need to create a comprehensive demo walkthrough for stakeholders.

**Objective:**

Generate a user-friendly walkthrough document that showcases what was built during the sprint.

**Instructions:**

1. **Review QA Results:**
   - Read `project_tracking/QA_REPORT.md`
   - Understand which features passed verification
   - Note any known issues or limitations

2. **Review Sprint Scope:**
   - Read the latest sprint file (e.g., `SPRINT_2.md`)
   - Identify all completed tasks
   - Understand the sprint goals and acceptance criteria

3. **Create Demo Walkthrough:**
   - Generate `project_tracking/DEMO_WALKTHROUGH.md`
   - Include:
     - **Sprint Summary**: What was built and why
     - **New Features**: List of new functionality with descriptions
     - **How to Use**: Step-by-step instructions for testing/using new features
     - **Technical Changes**: Backend APIs, frontend components, database changes
     - **Known Issues**: Any caveats or limitations from QA report
   - Use clear, non-technical language where possible
   - Include code examples or API endpoints where relevant

4. **Make it Stakeholder-Friendly:**
   - Focus on business value and user impact
   - Use screenshots or diagrams if applicable
   - Provide reproduction steps for demos
